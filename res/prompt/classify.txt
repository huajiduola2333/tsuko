#你要做的事情
请将以下通知数据根据指定分类规则进行归类，并补充每条通知的受众对象（audience）。最终以 JSON 格式输出，结构如下：

{
  "message": "输出执行结果，错误等“
  "summary": "请先写一段话，概括本批通知中对 ** 一开始提到的学生身份 ** 尤为重要的事项，如评优、招生、课程安排、校园活动等。",
  "notices": [
    {
      "title": "通知标题",
      "link": "通知链接",
      "date": "发布日期",
      "category": "所属类别序号（从下面选择，数据类型int）",
      "audience": "公告对象序号（从下面选择，数据类型int）"
    }
  ]
}

所属类别选项：
	1.	教学与学生管理（含招生、毕业、评优、学位授予、课程安排、社团、考试等）
	2.	科研与项目申报（含各类基金、课题、博士后考核、科研平台等）
	3.	会议与比赛活动（含讲座、培训、说明会、讲堂、比赛、展览等）
	4.	党建与行政事务（含党务、思政、评优、工作报送、制度建设等）
	5.	通知公告与后勤保障（含放假安排、安全通报、设施维护、校园服务等）
	6.	国际与校外交流合作（含国际项目、外派交流、合作通知等）

公告对象选项：
	1.	本科生（如果标题明确提及本科生）
	2.	研究生（如果标题明确提及研究生）
	3.	留学生（如果标题明确提及留学生）
	4.	学生（如果是面向学生且未提及具体学生类别）
	5.	教师（面向教师的事务）
	6.	全体师生（面向全体的事务）

请注意：
	•	每条通知都要合理判断其最贴切的分类类别（只选一个）和目标受众（也只选一个）。
	•	通知标题、链接、发布日期原样保留，不可更改。
	•	输出应为完整且规范的 JSON 格式。
